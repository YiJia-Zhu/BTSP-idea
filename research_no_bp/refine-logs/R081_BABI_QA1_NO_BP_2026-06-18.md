# R081 bAbI QA1 Pure No-BP Answer Selector

**Date**: 2026-06-18

## Purpose

Add a real QA training/evaluation task after the TinyStories statistical-cache concern. The goal is to test whether a pure no-BP neural mechanism can learn answer selection from context/question pairs, without BP, BPTT, pretrained encoders, API backbones, or test-answer updates.

## Implementation

Added:

- `babi_no_bp_qa_experiment.py`

Task:

- Dataset: `data/babi_qa_processed/en-qa1/`
- Train: 900 examples
- Validation: 100 examples
- Test: 1000 examples
- Output: one of six location answers

Compared systems:

| Method | Role | Stores raw text |
|---|---|---|
| `majority_no_memory` | no-memory baseline | no |
| `raw_lexical_retrieval` | raw retrieval diagnostic, violates no-raw-data | yes |
| `hashed_lookup_diagnostic` | statistical sparse feature-count diagnostic | no |
| `symbolic_location_tracker` | task-specific symbolic upper bound | no |
| `phase_dendritic_no_bp` | pure no-BP neural answer selector | no |

Pure no-BP mechanism:

- fixed random phase token features;
- branch features for question, full context, and question-subject context;
- local WTA target/wrong-winner updates;
- no gradients, no BP/BPTT, no pretrained representation;
- test answers are never used for updates.

## Commands

PYTHONDONTWRITEBYTECODE=1 python babi_no_bp_qa_experiment.py \

## Results

Smoke test split, 120 train / 120 test:

| Method | Test acc | Test CE | State bytes |
|---|---:|---:|---:|
| majority_no_memory | 0.158 | 1.800 | 162 |
| raw_lexical_retrieval | 0.225 | 1.778 | 39,143 |
| hashed_lookup_diagnostic | 0.158 | 16.821 | 6,723 |
| symbolic_location_tracker | 1.000 | 0.002 | 0 |
| phase_dendritic_no_bp | 0.817 | 1.115 | 5,021 |

Medium test split, 900 train / 1000 test:

| Method | Test acc | Test CE | State bytes |
|---|---:|---:|---:|

| hashed_lookup_diagnostic | 0.321 | 13.773 | 6,888 |
| symbolic_location_tracker | 1.000 | 0.002 | 0 |
| phase_dendritic_no_bp | 0.822 | 1.162 | 9,629 |

## Interpretation

R081 is a positive first QA result for the reset plan:

- `phase_dendritic_no_bp` strongly beats no-memory, raw retrieval, and hashed lookup on bAbI QA1 answer selection.
- The symbolic tracker confirms the task is solvable by an explicit state update, but it is a hand-coded upper bound, not the learning method.
- The sparse lookup diagnostic is much weaker than the neural no-BP learner, which helps address the concern that high scores only come from token-count statistics.

Boundaries:

- QA1 is single-supporting-fact location tracking; it is not multi-hop reasoning.
- The current phase/dendritic learner is still a small answer selector, not free-form generation.
- The fallback bAbI source does not include supporting fact ids, so supporting-fact supervision is not used.
